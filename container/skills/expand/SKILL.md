# Expand

Take compressed content (bullets, notes, outlines) and expand into full, polished prose.

## When to Use

- "Expand this..."
- "Flesh this out"
- "Write this out fully"
- "Turn these bullets into paragraphs"
- "Elaborate on this"

## Process

### 1. Understand the Input

- What format is the source? (bullets, outline, notes, fragment)
- What is the target? (full doc, email section, paragraph, message)
- What tone? Default to Shintaro's voice (see **persona** skill)
- How much expansion? (light, medium, full)

### 2. Analyze Content

- Identify the main points
- Understand logical relationships between points
- Note implicit context that should become explicit

### 3. Expand

For each point:
- Add context (why this matters)
- Add specifics (examples, details)
- Add transitions (connect to previous/next point)
- Maintain the original meaning exactly

### 4. Create Flow

- Opening that frames the content
- Paragraphs connect logically
- Vary sentence structure
- Conclusion if appropriate

## Expansion Depth

**Light (1.5-2x length):**
- Add transitions between points
- Minor context additions
- Good for: emails, messages, brief updates

**Medium (2-3x length):**
- Add context and reasoning
- Include examples
- Create paragraph structure
- Good for: doc sections, reports, proposals

**Full (3-5x length):**
- Comprehensive context
- Multiple examples
- Implications and next steps
- Full narrative flow
- Good for: articles, detailed write-ups

## Paragraph Structure

Each expanded paragraph should have:
1. **Topic sentence** -- the main point
2. **Supporting detail** -- evidence, examples, specifics
3. **Connection** -- link to the bigger picture or next point

## Shintaro's Voice (Default)

When expanding, reference the **persona** skill:
- Direct and warm, not corporate
- No em dashes
- Short paragraphs, white space
- Get to the point, then add depth
- Specific over generic

## Anti-Patterns

- Don't add fluff. Every sentence should add value
- Don't change meaning while adding context
- Don't over-expand simple points that don't need it
- Don't lose the original voice/tone
- Don't invent facts (note when examples are illustrative)
- Don't introduce AI-sounding language (see **de-ai** skill patterns)
